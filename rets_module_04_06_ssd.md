## 1. 개요

### 1.1 목적

본 문서는 RETS-MODULE-04의 주요 업무 화면을 도출하고, 각 화면의 구조·사용자 흐름·주요 UI 요소를 설계한다. 모든 화면은 `rets_02b_design_requirements.md`에 정의된 RETS 공통 디자인 시스템을 준수하여 구현한다.

### 1.2 공통 디자인 원칙 요약

| 항목 | 규칙 |
|------|------|
| 포인트 컬러 | `#F97316` (오렌지, 버튼·강조·선택 상태 등) |
| 모듈 뱃지 색상 | `#C2560B` (Pressed 톤) |
| 폰트 | `Noto Sans KR`, 최대 20px, 자간 `-0.04em` |
| 배경 (라이트) | `--bg: #FFFFFF`, `--surface: #F9FAFB` |
| 배경 (다크) | `--bg: #101111`, `--surface: #1c1d1d` |
| 헤더 스타일 | Blur Glass (`backdrop-filter: blur(14px) saturate(160%)`) |
| 팝업 | 금지 — 모달/드로어/Toast만 사용 |
| LLM 결과 | 자동 반영 금지 — 수락/거부 UI 필수 |
| Toast | 우측 하단, 최대 4개 동시 표시 |

### 1.3 화면 목록

| 화면 ID | 화면명 | 주요 탭/기능 | 연관 요구사항 |
|---------|--------|-------------|--------------|
| SSD-01 | 공통 레이아웃 (Header + 탭 구조) | 전체 공통 | DEV_NFR_003 |
| SSD-02 | 도메인 모델 탭 | 엔터티·관계·ER 다이어그램 | DEV_FR_002, DEV_FR_003 |
| SSD-03 | 용어 사전 탭 | 용어 목록·검색·등록 | DEV_FR_004 |
| SSD-04 | 요구사항 검토 탭 | 요구사항 목록·비표준 탐지 | DEV_FR_001, DEV_FR_005 |
| SSD-05 | LLM 결과 패널 (공통 컴포넌트) | 수락/거부 UI | DEV_FR_002~006 |
| SSD-06 | 완성도 검토 패널 | 개선 제안 목록 | DEV_FR_006 |

## 2. 공통 레이아웃 (SSD-01)

### 2.1 전체 화면 구조

```
┌──────────────────────────────────────────────────────────────────┐
│ HEADER (sticky, Blur Glass, h:52px)                              │
│  [RETS 뱃지] [도메인 모델링 및 용어 사전]  [로고]  [🌙][불러오기][내보내기▼] │
└──────────────────────────────────────────────────────────────────┘
┌──────────────────────────────────────────────────────────────────┐
│ WORKING TABS                                                     │
│  [ 도메인 모델 ]  [ 용어 사전 ]  [ 요구사항 검토 ]               │
└──────────────────────────────────────────────────────────────────┘
┌──────────────────────────────────────────────────────────────────┐
│ BODY (viewport 전체, 내부 패딩 30px)                             │
│                                                                  │
│  ←── 활성 탭에 따라 SSD-02 / SSD-03 / SSD-04 표시 ──→          │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 2.2 Header 상세 설계

**레이아웃**: 3-column flex 구조

| 영역 | 내용 | 스타일 |
|------|------|--------|
| 좌측 (`flex: 1`) | `[RETS]` 뱃지 + `도메인 모델링 및 용어 사전` | 뱃지 bg: `#C2560B`, 텍스트: white |
| 중앙 (`flex-shrink: 0`) | 조직 로고 이미지 | `logo_light.png` / `logo_dark.png` CSS 전환 |
| 우측 (`flex: 1`, 우측 정렬) | 버튼 그룹 | 아이콘 버튼 스타일 |

**우측 버튼 그룹 구성:**

```
[🌙 테마 전환]  [📂 불러오기]  [⬇ 내보내기 ▼]
```

- **테마 전환 버튼**: 아이콘(☀/🌙) 토글. 클릭 시 `data-theme` 속성 전환
- **불러오기 버튼**: `btn-secondary` 스타일. 클릭 시 숨겨진 `<input type="file" accept=".json">` 트리거
- **내보내기 버튼**: `btn-primary` 스타일 + 드롭다운 화살표. 클릭 시 메뉴 펼침

**내보내기 드롭다운 메뉴:**

```
┌─────────────────────────┐
│ 📄 용어 사전 (Markdown) │
│ 📊 용어 사전 (CSV)      │
│ ─────────────────────── │
│ 🗺 도메인 모델 (HTML)   │
│ 📝 도메인 모델 (Markdown)│
└─────────────────────────┘
```

**헤더 시각 스타일:**
```css
.header {
  background: var(--header-bg);          /* rgba(240,242,245,0.72) / rgba(14,35,35,0.80) */
  backdrop-filter: blur(14px) saturate(160%);
  border-bottom: 1px solid var(--border);
  position: sticky;
  top: 0;
  z-index: 100;
  height: 52px;
  padding: 0 30px;
}
```

### 2.3 Working Tabs 설계

```css
.tabs {
  display: flex;
  border-bottom: 1px solid var(--border);
  padding: 0 30px;
  background: var(--surface);
}
.tab-item {
  padding: 10px 16px;
  font-size: 13px;
  font-weight: 500;
  cursor: pointer;
  border-bottom: 2px solid transparent;
}
.tab-item.active {
  color: var(--brand-500);               /* #F97316 */
  border-bottom-color: var(--brand-500);
}
```

**탭 목록:**
- `도메인 모델` — 기본 진입 탭
- `용어 사전`
- `요구사항 검토`

### 2.4 Toast 컴포넌트 (공통)

```
우측 하단 고정 (position: fixed; bottom: 24px; right: 24px)
최대 동시 4개. 상태별 색상:
  success → border-left: 4px solid var(--success-500)
  error   → border-left: 4px solid var(--error-500)
  warning → border-left: 4px solid var(--warning-500)
  info    → border-left: 4px solid var(--info-500)
```

### 4.1 화면 구조 (2-column)

```
┌─────────────────────────────┬───────────────────────────────────┐
│ LEFT PANEL (40%)            │ RIGHT PANEL (60%)                 │
│                             │                                   │
│ 🔍 [검색창: 용어 검색...]   │ (용어 선택 시 상세 패널 표시)    │
│                             │                                   │
│ [🤖 용어 정의 생성]         │ 용어명: 요구사항                  │
│ [⚠ 중복 검사]               │ ─────────────────────────────   │
│ [+ 용어 추가]               │ 정의: 시스템이 수행해야 하는...   │
│                             │ 동의어: 소프트웨어 요구사항, ...  │
│ ── 용어 목록 ──             │ 관련 엔터티: 요구사항 →[이동]   │
│ • 요구사항      active ●    │ 출처: LLM                        │
│ • 이해관계자    active ●    │ 비고: -                          │
│ • 도메인 모델   candidate ○ │                                   │
│ • 용어 사전     active ●    │     [편집]  [삭제]               │
│ ⚠ 기능 명세    (충돌)       │                                   │
└─────────────────────────────┴───────────────────────────────────┘
```

### 4.2 용어 목록 아이템 상태 표시

| 상태 | 배지 스타일 | 색상 |
|------|-----------|------|
| `active` | `badge-success` | 초록 |
| `candidate` | `badge-warning` | 노랑 |
| `deprecated` | `badge-neutral` | 회색 |
| 충돌 감지 | `badge-warning` + ⚠ 아이콘 | 노랑 경고 |

### 4.3 용어 검색

- 입력 즉시 실시간 필터링 (debounce 150ms)
- 검색 범위: `term`, `definition`, `synonyms` 전체 텍스트
- 결과 없음: Empty 상태 UI ("검색 결과가 없습니다")

### 4.4 중복·충돌 탐지 결과 표시

```
┌───────────────────────────────────────────────────────────────────┐
│ ⚠️  중복·충돌 용어 탐지 결과 (2쌍 발견)                          │
│                                                                   │
│ [충돌 1] '요구사항' ↔ '기능 명세'                                 │
│ 이유: 동의어 중복 ("소프트웨어 요구사항")                          │
│ → 기준 용어 선택: ( ● 요구사항 ) ( ○ 기능 명세 ) [통합]          │
│                                                                   │
│ [충돌 2] '도메인 모델' ↔ '도메인 모형'                            │
│ 이유: 완전 일치 유사어                                             │
│ → 기준 용어 선택: ( ● 도메인 모델 ) ( ○ 도메인 모형 ) [통합]     │

```

## 6. LLM 결과 패널 (SSD-05) — 공통 컴포넌트

**사용 위치**: 엔터티 추출, 관계 추천, 용어 정의 생성, 완성도 검토 결과 표시 시

**화면 우측 슬라이드인 드로어 또는 Body 내 인라인 패널로 표시**

### 6.1 LLM 로딩 상태

```
┌─────────────────────────────────────────────┐
│ 🤖 LLM 분석 중                         [×] │
│                                             │
│    ⠿ 요구사항을 분석하여 도메인 엔터티      │
│      후보를 추출하고 있습니다...            │
│                                             │
│    [취소]                                   │
└─────────────────────────────────────────────┘
```

### 6.2 엔터티 후보 결과 표시

```
┌─────────────────────────────────────────────────────────────────┐
│ 🤖 LLM 엔터티 추출 결과 (5개 후보)    [전체 수락] [전체 거부] │
│                                                                 │
│ 1. 📦 요구사항                                [✓ 수락] [✗ 거부] │
│    설명: 시스템이 수행해야 하는 기능 또는 제약                  │
│    속성: req_id(PK), req_name, priority, status                 │
│                                                                 │
│ 2. 📦 이해관계자                              [✓ 수락] [✗ 거부] │
│    설명: 요구사항에 관여하는 모든 당사자                         │
│    속성: id(PK), name, role, organization                       │
│                                                                 │
│ 3. 📦 도메인 모델                             [✓ 수락] [✗ 거부] │
│    설명: 도메인의 핵심 개념과 관계를 시각화한 모델              │
│    속성: id(PK), name, version                                  │
└─────────────────────────────────────────────────────────────────┘
```

**수락 버튼**: `btn-primary` 스타일 (포인트 컬러)  
**거부 버튼**: `btn-danger` 스타일

### 6.3 관계 후보 결과 표시

```
┌─────────────────────────────────────────────────────────────────┐

│                                                                 │
│ 2. 용어 사전 ||--|{ 요구사항                                   │
│    유형: association   레이블: "포함"                           │
│    카디널리티: 용어 사전(1) : 요구사항(1..*)                   │
│                               [✓ 수락] [✗ 거부]                │
└─────────────────────────────────────────────────────────────────┘
```
