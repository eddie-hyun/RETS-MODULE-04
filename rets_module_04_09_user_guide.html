RETS-MODULE-04 — 도메인 모델링 및 용어 사전 구축 도구

요구사항 문서로부터 도메인 엔터티·관계·용어를 LLM으로 자동 추출하고, 이해관계자 간 공통 이해를 지원하는 도메인 모델 및 용어 사전을 구축할 수 있는 단일 페이지 웹 도구입니다.

버전 1.0 발행일 2026-05-07 대상 요구사항 분석가 / 아키텍트 / PM 소요 시간 5–15분 (첫 실행)

1개요

본 도구는 단일 HTML 파일(rets_module_04_07_app.html)로 구성되어 있어 별도 서버 없이 브라우저에서 직접 실행됩니다. LLM 릴레이 서버를 통해 도메인 엔터티 추출·관계 추천·용어 정의 생성·완성도 검토 같은 지능화 기능을 제공하며, 모든 사용자 작업은 수락/거부 UI를 거치므로 LLM 결과를 무비판적으로 반영하지 않습니다.

주요 기능 요약

📥 요구사항 로드
DEV REQ JSON 파일을 불러와 요구사항 목록을 화면에 표시합니다.
🤖 LLM 자동 추출
요구사항을 분석하여 엔터티·관계·용어 정의를 자동 생성합니다.
🗺 ER 다이어그램
Mermaid erDiagram으로 도메인 모델을 시각화합니다.
📖 용어 사전
표준 용어를 등록·검색하고 동의어/관련 엔터티를 관리합니다.
⚠ 중복·비표준 탐지
용어 충돌과 요구사항 내 비표준 용어를 자동으로 찾아냅니다.
📤 다양한 내보내기
용어 사전(MD/CSV)과 도메인 모델(HTML/MD)을 파일로 출력합니다.
ℹ 본 도구의 위치

RETS PoC 산출물 흐름에서 본 도구(MD07)는 STEP 3 단계의 실행 가능 모듈에 해당하며, 업스트림으로는 SRS·DDD·SSD를, 다운스트림으로는 벤치마크 결과(MD08)와 본 사용자 가이드(MD09)를 갖습니다.

준비물

  • DEV REQ JSON 파일 — RETS 공통 스키마(rets_02c_req_schema.json)를 준수해야 합니다.
  • team_id — LLM 릴레이 서버에서 발급된 팀 식별자(1–64자). 헤더 우측 입력란에 입력합니다.
  • 실행 파일 — rets_module_04_07_app.html 파일 한 개.
⚠ team_id가 없으면?

LLM 릴레이 서버에 접속할 수 없어 자동 추출 기능(엔터티/관계/용어/완성도)이 동작하지 않습니다. 수동 추가/편집은 가능합니다.

3첫 실행 — 5분 튜토리얼

샘플 DEV REQ 데이터를 로드해 도메인 모델을 만들고 용어 사전을 내보내기까지의 흐름을 따라가 보세요.

  1. 앱 열기
    rets_module_04_07_app.html 파일을 더블클릭하거나 브라우저로 드래그합니다. 우측 상단의 [↗ 앱 실행] 버튼으로도 열 수 있습니다.
  2. team_id 입력
    헤더 우측 team_id 입력란에 발급받은 식별자를 입력합니다 (예: team-04).
  3. DEV REQ JSON 로드
    [↑ 불러오기] 버튼 → 파일 선택. 로드 성공 시 우측 하단에 Toast 알림이 표시됩니다.
  4. 엔터티 자동 추출
    [도메인 모델] 탭 → [🤖 엔터티 추출] 클릭. LLM 응답이 도착하면 우측 드로어에 후보 목록이 펼쳐집니다.
  5. 엔터티 수락/거부
    각 후보의 [수락]/[거부] 버튼으로 결정합니다. [전체 수락]도 가능합니다.
  6. 관계 자동 추천
    [🤖 관계 추천] 클릭 → 후보를 검토 후 수락하면 ER 다이어그램이 즉시 갱신됩니다.
  7. 용어 정의 생성
    [용어 사전] 탭 → [🤖 용어 정의 생성] 클릭. 결과를 수락하면 사전에 자동 등록됩니다.
  8. 내보내기
    [↓ 내보내기] 메뉴 → [용어 사전 (Markdown)] 또는 [도메인 모델 (HTML)] 선택. 즉시 다운로드됩니다.
✓ 5분이면 충분합니다

샘플 요구사항이 20–30개라면 LLM 호출 3회와 수동 검토만으로 도메인 모델 초안이 완성됩니다.

5핵심 작업 흐름

실무에서 가장 자주 수행하는 8가지 작업을 정리했습니다.

5.1 요구사항 불러오기

RETS 공통 스키마를 따르는 DEV REQ JSON을 로드합니다.

  1. [↑ 불러오기] 클릭
    파일 선택 다이얼로그가 열립니다.
  2. JSON 파일 선택
    스키마(req_id, req_name, req_type 등)가 검증됩니다.
  3. 로드 결과 확인
    총 N개 요구사항 로드 / 오류 항목 스킵 정보가 Toast로 안내됩니다.
⚠ 스키마 오류

req_id 누락이나 패턴 불일치(^DEV_(FR|NFR)_\d{3}(_\d{2}){0,4}$) 항목은 자동으로 스킵되며 Toast로 갯수가 표시됩니다.

5.2 엔터티 추출·확정

로드된 요구사항을 LLM이 분석하여 도메인 엔터티 후보를 자동 추출합니다.

  1. [🤖 엔터티 추출] 클릭
    최초 클릭 시 LLM 세션이 자동 생성됩니다(team_id 필요).
  2. 후보 검토
    우측 드로어에 5–10개의 엔터티 후보가 이름·설명·속성과 함께 표시됩니다.
  3. 수락 / 거부
    개별 [수락] 또는 [전체 수락]. 수락된 엔터티만 도메인 모델에 반영됩니다.
  4. 직접 추가·편집
    [+ 엔터티 추가] 또는 목록의 [편집] 버튼으로 수동 보완.

5.3 관계 추천·확정

확정된 엔터티들 사이의 관계를 LLM이 추천합니다(엔터티 2개 이상 필요).

  1. [🤖 관계 추천] 클릭
  2. 후보 검토
    소스/타겟·유형(association, aggregation 등)·카디널리티·레이블이 표시됩니다.
  3. 수락 시 즉시 ER 반영
    우측 ER 다이어그램이 자동 재렌더링됩니다.

5.4 용어 사전 구축

  1. [용어 사전] 탭으로 이동
  2. [🤖 용어 정의 생성] 클릭
    엔터티별 비즈니스 용어 정의 후보가 생성됩니다.
  3. 수락하여 사전 등록
  4. 검색 / 상세 조회
    검색창에 키워드 입력 시 실시간 필터링(150ms debounce).
  5. 관련 엔터티 점프
    상세 패널의 [📦 엔터티명] 태그를 클릭하면 도메인 모델 탭의 해당 엔터티로 이동합니다.

5.5 중복·충돌 통합

완전 일치 또는 동의어 중복으로 인한 충돌 용어를 탐지하고 단일 표준으로 병합합니다.

  1. [⚠ 중복 검사] 클릭
    충돌 쌍 목록이 상단 배너로 표시됩니다.
  2. 기준 용어 선택
    두 용어 중 하나를 기준으로 [통합] 버튼 클릭.
  3. 병합 결과
    동의어와 관련 엔터티가 합쳐지고, 다른 용어는 사전에서 제거됩니다.

5.6 비표준 용어 정제

용어 사전의 동의어를 기준으로 요구사항 텍스트에 사용된 비표준 표현을 찾아 표준 용어로 교체합니다.

  1. [요구사항 검토] 탭 이동
  2. [⚠ 비표준 용어 검사] 클릭
    탐지된 N건이 Toast로 안내되고 해당 행이 노란 배경으로 표시됩니다.
  3. 개별 [수락] 또는 [전체 수락]
    선택한 항목의 텍스트가 표준 용어로 즉시 치환됩니다.

5.7 완성도 검토

현재까지 만든 도메인 모델과 용어 사전 전체를 LLM이 진단하고 누락 항목을 제안합니다.

  1. [🔍 완성도 검토] 클릭
  2. 제안 카테고리별 검토
    누락 엔터티 / 부족한 관계 / 미등록 용어 / 불완전한 정의로 그룹화됩니다.
  3. 제안 [추가] 또는 [무시]
    [추가]를 누르면 도메인 모델 또는 사전에 즉시 반영됩니다.

실무 팁

  • 처음에는 요구사항 50건 이하로 시작하면 LLM 응답 속도가 더 빠릅니다.
  • 엔터티 후보 중 너무 추상적인 항목(예: "시스템", "데이터")은 거부하는 편이 모델 가독성에 도움됩니다.
  • 관계 추천은 엔터티가 3개 이상 확정된 뒤에 실행하면 더 정확합니다.
  • 비표준 용어 검사 전에 용어 사전의 동의어를 충실히 기재하세요. 동의어가 곧 탐지 룰이 됩니다.
  • 중요한 작업 전에 [내보내기]로 백업해 두면 안전합니다 — 본 도구는 브라우저 메모리·localStorage 기반입니다.
  • 다크 모드는 ER 다이어그램에 자동 적용됩니다(filter: invert + hue-rotate).
ℹ 자동 저장

도메인 모델·용어 사전·테마/언어/team_id 설정은 브라우저 localStorage에 자동 저장되어 다음 방문 시 복원됩니다. 단, 브라우저 캐시를 비우면 로컬 데이터가 사라지므로 주기적인 내보내기를 권장합니다.

7문제 해결

증상별 빠른 진단

증상원인해결
[엔터티 추출] 클릭 시 team_id 필요 Toast team_id 미입력 헤더 우측의 team_id 입력란에 식별자 입력 후 재시도.
LLM 호출 실패 / 상태 바 오류 네트워크 단절 또는 릴레이 서버 일시 장애 잠시 후 재시도. 지속되면 팀 관리자에게 team_id 유효성 확인 요청.
JSON 로드 후 항목 일부가 누락 스키마 오류 항목이 자동 스킵됨 Toast의 스킵 갯수 확인. 원본 JSON에서 req_id 패턴/필드 점검.
ER 다이어그램이 코드 블록으로 표시 Mermaid 렌더링 실패(특수문자 등) 엔터티명에 한글·영문·숫자·언더스코어 외 특수문자 사용 금지. 다른 엔터티명으로 변경 후 재시도.
관계 추천이 빈 결과 확정 엔터티가 2개 미만 엔터티를 2개 이상 확정한 후 재시도.
비표준 용어가 탐지되지 않음 용어 사전에 동의어가 등록되지 않음 표준 용어의 [편집] → 동의어 필드에 비표준 표현을 추가.
다운로드 동작이 차단됨 브라우저 다운로드 권한 차단 주소창 다운로드 차단 아이콘 클릭 → 항상 허용으로 설정.

로그 확인

브라우저 개발자 도구(F12)의 Console 탭에서 LLM 관련 오류 메시지를 확인할 수 있습니다. Network 탭에서는 https://ai-relay.mooo.com:11443 에 대한 요청·응답을 추적할 수 있습니다.

8FAQ

오프라인에서도 사용할 수 있나요?
수동 추가·편집·내보내기·로컬 저장은 오프라인에서도 동작합니다. 단, LLM 자동 추출 기능과 Mermaid CDN 렌더링은 인터넷 연결이 필요합니다.
여러 프로젝트의 데이터를 동시에 관리할 수 있나요?
현재는 단일 프로젝트 컨텍스트만 지원합니다. 다른 프로젝트로 전환하려면 [내보내기]로 백업 후 페이지 새로고침을 권장합니다.
LLM 결과가 만족스럽지 않으면 어떻게 하나요?
모든 LLM 결과는 [수락]/[거부] 검토 단계를 거치므로 불만족스러운 후보는 거부 후 [엔터티 추가]·[관계 추가]·[용어 추가]로 직접 보완할 수 있습니다.
같은 team_id를 다른 사람과 공유해도 되나요?
팀당 활성 LLM 세션은 1개로 제한됩니다(릴레이 서버 정책). 동시 사용 시 마지막 세션 생성자가 우선되며 다른 멤버는 일시적으로 호출 실패할 수 있습니다.
Mermaid 다이어그램을 수정할 수 있나요?
엔터티/관계 데이터를 변경하면 다이어그램이 자동 재생성됩니다. 직접 Mermaid 코드를 편집하려면 [도메인 모델 (Markdown)] 내보내기 후 외부 에디터에서 작업하세요.
다국어는 어디까지 지원되나요?
현재 UI 레이블은 한국어/영어 두 언어를 지원합니다(헤더 [한/En] 버튼). 사용자가 입력·생성한 데이터(엔터티명·정의 등)는 자동 번역되지 않습니다.
iframe 안에 임베드해도 되나요?
예. 단일 HTML 파일이고 자체 완결 구조이므로 RETS 통합 앱에서 <iframe> 으로 삽입할 수 있도록 설계되었습니다.
