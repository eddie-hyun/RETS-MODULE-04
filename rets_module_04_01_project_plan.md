## 5. 협업 방식

- **문서 관리**: 모든 산출물은 `/RETS-MODULE-04/` 루트 디렉토리에 지정된 파일명으로 저장
- **요구사항 추적**: SRS(MD03)를 기준으로 TDD, DDD, SSD와의 추적 컬럼 유지
- **코드 리뷰**: 단일 HTML 파일 구현 후 팀 내 교차 검토 수행
- **LLM 활용**: AI 기반 기능 구현 시 LLM 릴레이 서버 가이드(`rets_02a_llm_relay_server_dev_guide.md`) 준수
- **디자인 준수**: 구현 전 `rets_02b_design_requirements.md` 및 디자인 시스템 HTML 참고
- **개발 방법론**: TDD(Test-Driven Development) — 요구사항 명세 단계에서 테스트 케이스를 선 도출 후 개발

## 6. 상위 수준 모듈 개발 요구사항

본 모듈에서 개발해야 하는 핵심 기능은 RFP 2.3.4절 기반으로 다음과 같이 정의한다.

### 6.1 기능 요구사항 (요약)

| # | 요구사항 | 구분 | 우선순위 |
|---|----------|------|---------|
| F-01 | 개발 요구사항(DEV REQ) JSON 파일 불러오기 및 파싱 | 기능 | MUST |
| F-02 | LLM 기반 핵심 도메인 엔터티 자동 추출 및 사용자 검토·확정 | 기능 | MUST |
| F-03 | LLM 기반 엔터티 간 관계(Relationship) 자동 추천 및 사용자 확정 | 기능 | MUST |
| F-04 | Mermaid ER 다이어그램 기반 도메인 모델 시각화 | 기능 | MUST |
| F-05 | 사용자 직접 엔터티·관계 추가·수정·삭제 | 기능 | MUST |
| F-06 | LLM 기반 용어 정의 자동 생성 및 용어 사전 등록 | 기능 | MUST |
| F-07 | 중복·충돌 용어 탐지 및 통합 | 기능 | MUST |
| F-08 | 용어 사전 검색 및 상세 조회 | 기능 | MUST |
| F-09 | 용어 사전 기반 요구사항 내 비표준 용어 탐지 및 수정 제안 | 기능 | SHOULD |
| F-10 | LLM 기반 도메인 모델 완성도 검토 및 개선 제안 | 기능 | MUST |
| F-11 | 용어 사전(Markdown/CSV) 및 도메인 모델 다이어그램(HTML/Markdown) 내보내기 | 기능 | MUST |
| F-12 | 한글/영어 다국어 지원 | 기능 | MUST |

## 7. 리스크 및 대응 계획

| # | 리스크 | 발생 가능성 | 영향도 | 대응 방안 |

| R-03 | Mermaid ER 다이어그램 렌더링 이슈 | 저 | 중 | CDN 방식 Mermaid 라이브러리 사용. 렌더링 실패 시 텍스트 형식 fallback 제공 |
| R-04 | JSON 파일 파싱 오류 (스키마 불일치) | 중 | 중 | 로드 시 스키마 유효성 검증 로직 구현, 오류 항목 Toast 알림 처리 |

| R-06 | 시간 제약으로 인한 일부 기능 미완성 | 상 | 중 | MoSCoW 기반 우선순위 적용. MUST 기능 선행 구현, SHOULD/COULD는 이후 반영 |
| R-07 | 다국어 지원 구현 누락 | 저 | 저 | i18n 객체 구조로 초기 설계하여 언어 전환 기능 간단히 구현 가능하도록 준비 |
