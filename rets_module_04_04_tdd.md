## 1. 개요

### 1.1 목적

본 문서는 RETS-MODULE-04의 SRS(소프트웨어 요구사항 명세서)에 정의된 기능·비기능 요구사항을 기반으로, PoC 단계에서 검증해야 할 테스트 시나리오와 테스트 케이스를 정의한다. TDD(Test-Driven Development) 방법론에 따라 요구사항 명세 단계에서 테스트를 선 도출한다.

### 1.2 테스트 케이스 형식

각 테스트 케이스는 **Given-When-Then** 형식으로 작성한다.

- **Given**: 사전 조건 (테스트 실행 전 상태)
- **When**: 사용자 액션 또는 이벤트
- **Then**: 기대 결과

## 2. 핵심 사용자 시나리오 (KUS)

핵심 사용자 시나리오는 모듈의 주요 워크플로우를 처음부터 끝까지 검증하는 E2E 시나리오이다.

### KUS-01. 도메인 모델링 완전 워크플로우

**목적**: DEV REQ JSON 로드 → 엔터티 추출 → 관계 추천 → 용어 사전 구축 → 내보내기까지 전체 흐름 검증

**사전 조건**: 브라우저에서 `rets_module_04_07_app.html` 열기 완료, 유효한 DEV REQ JSON 파일 준비

**시나리오 단계**:
1. Header의 [불러오기] 버튼 클릭 → DEV REQ JSON 파일 선택 및 로드
2. 요구사항 목록 그리드 확인
3. [엔터티 추출] 버튼 클릭 → LLM 호출 → 엔터티 후보 목록 확인 → 수락/거부 처리
4. [관계 추천] 버튼 클릭 → LLM 호출 → 관계 후보 확인 → 수락 처리 → ER 다이어그램 확인
5. [용어 정의 생성] 버튼 클릭 → LLM 호출 → 용어 정의 확인 → 수락 → 용어 사전 등록
6. 용어 사전 탭에서 검색 및 상세 조회
7. [완성도 검토] 버튼 클릭 → 개선 제안 확인 → 일부 수락
8. Header [내보내기] → 용어 사전 Markdown 다운로드 확인

**기대 결과**: 각 단계가 오류 없이 수행되고, 최종 용어 사전 파일이 다운로드됨

## 4. 테스트 실행 가이드

### 4.1 Playwright 자동화 테스트 시나리오 (KUS-01)

다음 Playwright 스크립트 구조를 기반으로 KUS-01 핵심 시나리오를 자동화한다.

```javascript
// playwright_kus01.spec.js
const { test, expect } = require('@playwright/test');

test('KUS-01: 도메인 모델링 완전 워크플로우', async ({ page }) => {
  // 1. 앱 열기
  await page.goto('file:///path/to/rets_module_04_07_app.html');
  await expect(page).toHaveTitle(/RETS-MODULE-04/);

  // 2. team_id 설정 (필요 시)
  // await page.fill('#team-id-input', 'team04');

  // 3. JSON 파일 불러오기
  const fileInput = page.locator('input[type="file"]');
  await fileInput.setInputFiles('./benchmark/dev_req_sample.json');
  await expect(page.locator('.req-grid')).toBeVisible();

  // 4. 엔터티 추출 버튼 클릭
  await page.click('#btn-extract-entities');
  await expect(page.locator('.loading-spinner')).toBeVisible();
  await expect(page.locator('.entity-candidates')).toBeVisible({ timeout: 30000 });

  // 5. 첫 번째 엔터티 수락
  await page.click('.entity-candidate:first-child .btn-accept');

  // 6. 용어 사전 탭 이동
  await page.click('[data-tab="glossary"]');
  await expect(page.locator('.glossary-list')).toBeVisible();

  // 7. 내보내기
  await page.click('#btn-export');
  const [download] = await Promise.all([
    page.waitForEvent('download'),
    page.click('.export-option-markdown'),
  ]);
  expect(download.suggestedFilename()).toMatch(/glossary_.*\.md/);
});
```
